## 핵심 아이디어

**단일 실제 주행 로그**를 신경망 기반 센서 시뮬레이터로 변환하여 행위자 추가/재배치, 반사실적(counterfactual) 희귀 시나리오를 리얼리스틱 센서 데이터로 생성한다.

### 핵심 기능

```
실제 주행 로그 (LiDAR + 카메라)
    │
    ▼  Neural Reconstruction
암묵적 신경 표현 (NeRF 기반 씬 표현)
    │
    ├─ 행위자 추가: 새로운 차량/보행자 삽입
    ├─ 행위자 재배치: 기존 에이전트 위치/속도 변경
    ├─ 날씨 변환: 맑음 → 비/안개 (제한적)
    ├─ 자아 궤적 변경: 다른 경로로 주행 시뮬레이션
    │
    ▼
반사실적 시나리오의 리얼리스틱 센서 데이터
```

### 반사실적 시나리오 예시

```
원본: 빈 고속도로 순항
변환: 앞에 급정거하는 트럭 추가 → ego 충돌 회피 반응 시뮬레이션

원본: 맑은 낮 교차로
변환: 우측에서 달려오는 자전거 삽입 → 교차로 충돌 위험 시나리오
```

## 장단점

**장점**
- 기존 로그 하나에서 수십 가지 희귀 시나리오 변형 생성 가능
- 리얼리스틱한 센서 데이터 (NeRF 기반) → 도메인 갭 최소화
- 폐루프(closed-loop) 테스트에 직접 활용 가능

**단점**
- LiDAR + 카메라 원본 로그 필요 (캡션만으로는 사용 불가)
- NeRF 학습 비용이 높음 (로그당 수 시간)
- 현재는 날씨/조명 변환에 제한적

## 프로젝트 적용 포인트

### Gap-4 / Gap-6 → EXP-002 Phase D

Phase A에서 식별된 희귀 시나리오를 기존 로그의 반사실적 변조로 보완:

| 식별된 갭 시나리오 | UniSim 변조 방법 |
|----------------|----------------|
| "안개 야간 고속도로 역주행" | 고속도로 로그 + 역주행 차량 삽입 |
| "교량 위 빗길 급제동" | 교량 로그 + 급제동 에이전트 재배치 |
| "보행자 돌발 횡단 + 야간" | 야간 로그 + 보행자 궤적 변조 |

프로젝트에 raw 센서 로그가 있다면:
1. Phase A: 희귀 시나리오 클러스터 식별
2. 가장 유사한 기존 로그 선택
3. UniSim으로 행위자 추가/재배치
