## 핵심 아이디어

VLM + LLM을 활용해 **문제 식별 → 데이터 큐레이션 → 자동 레이블링 → 시나리오 생성 검증**의 반복 루프를 구성한다. 희귀 카테고리(long-tail)를 자동 발견하고 처방하는 엔드-투-엔드 파이프라인이다.

### AIDE 4단계 반복 루프

```
1. 문제 식별 (Problem Identification)
   VLM으로 현재 모델이 실패하는 케이스를 자동 분석
   → "안개 + 야간 조건에서 보행자 탐지 실패율 높음"

2. 데이터 큐레이션 (Data Curation)
   LLM이 실패 케이스 설명을 쿼리로 변환 → 유사 데이터 검색
   → 희귀 카테고리에 해당하는 클립 수집

3. 자동 레이블링 (Auto-Labeling)
   LLM이 수집된 데이터에 자동으로 레이블 부여
   → 인간 레이블러 개입 최소화

4. 시나리오 생성 검증 (Scenario Generation & Validation)
   생성된 시나리오로 모델 재학습 후 성능 검증
   → 개선 확인 시 루프 반복, 아니면 데이터 큐레이션 재조정
```

### 분류체계 생성 방식

```python
# AIDE 방식: 샘플 캡션 → LLM → 분류체계
system_prompt = """
아래 1,000개 운전 장면 캡션을 분석하여 
20~50개의 시나리오 카테고리 체계를 제안하라.
각 카테고리는 상호 배타적이어야 하며,
희귀하지만 중요한 시나리오를 별도 카테고리로 분리하라.
"""
taxonomy = gpt4o(system_prompt + sample_captions)
```

## 장단점

**장점**
- 인간 개입 없이 long-tail 시나리오 자동 발견 및 처방
- 반복 루프로 점진적 커버리지 향상 가능
- VLM + LLM 조합으로 다양한 모달리티(텍스트, 이미지) 적용 가능

**단점**
- VLM 추론 비용이 높음 (이미지 기반)
- 텍스트 캡션만 있는 경우 VLM 사용 제한
- 반복 루프의 수렴 기준 설정 필요

## 프로젝트 적용 포인트

### Gap-3 / Gap-4 / Gap-6 → EXP-002 Phase B

AIDE의 핵심 아이디어를 캡션 기반 파이프라인에 적용:

```
1. 문제 식별: Phase A 클러스터링 결과에서 소규모 클러스터 = 취약 시나리오
2. 데이터 큐레이션: 취약 시나리오 쿼리 → bge-m3 검색으로 유사 클립 수집
3. 분류체계 생성: 1,000개 캡션 샘플 → GPT-4o → 20~50개 카테고리 taxonomy
4. 전체 분류: 299k 캡션 → GPT-4o-mini → 카테고리 할당
5. 반복: 희귀 카테고리 식별 → 수집 처방 → 재분류
```

비용 추정: ~$5~10 (전체 299k 분류 기준)
