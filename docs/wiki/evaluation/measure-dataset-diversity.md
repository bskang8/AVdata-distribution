## 핵심 아이디어

135개 공개 데이터셋을 분석한 결과, **"diversity"라는 단어가 실증적 근거 없이 선언적으로 사용**되고 있음을 폭로한다. 사회과학의 **측정 이론(measurement theory)**에 기반하여 다양성 정량화의 원칙을 제시한다.

### 다양성 측정의 4가지 원칙

1. **구성 타당성 (Construct Validity)**: 측정 지표가 실제로 "다양성"을 측정하는가?
2. **신뢰성 (Reliability)**: 동일 데이터에 재적용했을 때 일관된 결과가 나오는가?
3. **비교 가능성 (Comparability)**: 서로 다른 데이터셋 간 비교가 가능한가?
4. **검증 가능성 (Verifiability)**: 제3자가 결과를 재현할 수 있는가?

### 흔한 선언적 다양성의 문제

```
예시 (잘못된 방식):
"fog의 visibility는 0.12이다" → 근거 없는 수동 할당
"rain(0.65) vs snow(0.72)" → 차이 0.07의 근거 불명확

이 프로젝트의 ODD 연속 벡터 방식이 해당 논문이 비판하는 행태의 전형이다.
```

### 권장 측정 방식

- 이산 카테고리 빈도 → 직접 검증 가능한 수치
- 임베딩 공간 클러스터 분포 → 데이터 자체에서 도출
- Vendi Score, Metric Space Magnitude 등 수학적으로 정의된 지표

## 프로젝트 적용 포인트

### EXP-002 방법론 정당화 근거

이 논문은 현재 파이프라인의 `fog=0.12` 방식이 **방법론적 설계 오류**임을 학술적으로 뒷받침하는 핵심 근거다:

```
기존: fog → 0.12 (도메인 직관, 검증 불가)
     rain(0.65) vs snow(0.72) 차이 0.07의 근거 없음

대안: HDBSCAN 클러스터 빈도 (직접 검증 가능)
     이산 ODD 조합 카운트 (fog × night × highway = 89개)
```

이 논문의 원칙을 인용하면 임베딩 클러스터링 방식으로의 전환이 임의적 선택이 아닌 **측정 이론에 근거한 필연적 선택**임을 정당화할 수 있다.
