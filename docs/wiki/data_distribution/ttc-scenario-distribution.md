## 핵심 아이디어

실제 교통 밀도(traffic density)를 입력으로 **TTC(Time-to-Collision) 분포**를 예측하는 확률 모델.  
테스트 시나리오 선택 시 "얼마나 현실적인가?"를 정량화하여 불필요한 테스트를 제거한다.

### TTC 분포 모델

**기반**: Poisson 분포 (교통 참여자의 독립적 무작위 도착)

```
P(intl ≤ t) = 1 - e^{-λ_v × t}   # 차량 도착 간격

P(TTC ≤ t) = (1 - e^{-λ_p × t}) × Σ P(intl=T) × (t/T)
```

- `λ_v`: 차량 평균 도착률 (대/초)
- `λ_p`: 보행자 평균 도착률 (명/초)
- **TTC < 3초**: 충돌 위험 임계값

### 검증 결과 (링쾨핑, 스웨덴)

| Date/Hour | 보행자/차량 수 | TTC <3s 실제 | 예측 |
|-----------|-------------|-------------|------|
| 17/07 07:00 | 67/332 | 0.01 | 0.14 |
| 05/08 15:00 | 77/115 | 0.02 | 0.08 |

→ 실제 분포 < 모델 예측 (worst-case 보수적 추정)

### 테스트 시나리오 선택 기여 (기존 대비)

| TTC 구간 | 모델 | Uniform | Normal |
|---------|------|---------|--------|
| 0~1s | 33 | 100 | 6 |
| 1~3s | 150 | 200 | 105 |
| 3~5s | 235 | 198 | 402 |
| 5~10s | 582 | 502 | 486 |

→ 모델이 critical scenario (TTC <3s)에 더 많은 테스트 할당

## 장단점

**장점**
- 수학적으로 단순 (Poisson 분포) → 해석 가능
- 교통 밀도만 입력으로 → 추가 데이터 수집 불필요
- Worst-case 보수적 추정 → 안전 마진 제공

**단점**
- 보행자/차량의 독립성 가정 → 실제 상호작용 미반영
- 교차로 외 시나리오 (고속도로, 날씨 등) 미포함
- 시공간적 특성 (도시별, 시간대별) 차이 반영 부족

### Gap-4 (분포 편향) 부분 적용

현재 83k 클립의 시나리오 분포를 TTC 기반으로 분석 가능.  
단, 직접 TTC 계산보다는 **핵심 아이디어 — 현실적 분포 반영한 테스트 할당**을 차용.

**적용 아이디어:**
```python
# 각 쿼리 유형별 실제 발생 빈도 추정
query_frequencies = {
    "pedestrian_crossing": 0.30,  # urban 교차로에서 흔함
    "fog_driving": 0.02,          # 희귀
    "school_zone": 0.05,
}

# 빈도 역수로 희귀 시나리오에 더 많은 검색 예산 할당
weights = {k: 1/v for k, v in query_frequencies.items()}
```
