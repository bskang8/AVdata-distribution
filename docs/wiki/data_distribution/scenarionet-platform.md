## 핵심 아이디어

Waymo / nuScenes / Lyft / nuPlan 등 이기종 데이터셋을 **통합 시나리오 표현으로 변환**하고, 실세계 데이터와 합성 데이터 간의 커버리지 갭을 시나리오 클러스터링으로 시각화하는 오픈소스 플랫폼이다.

### 26가지 시나리오 분류 체계

ScenarioNet은 교통 시나리오를 다음 기준으로 분류한다:

| 대분류 | 예시 |
|--------|------|
| 도로 구조 | 교차로, 라운드어바웃, 고속도로, 병목 |
| 에이전트 유형 | 차량, 보행자, 자전거, 오토바이 |
| 상호작용 패턴 | 앞지르기, 차선 변경, 양보, 추월 |
| 위험 수준 | 위험 없음, 근접 추월, 급제동, 충돌 직전 |

총 26개 시나리오 카테고리로 이기종 데이터셋의 커버리지 차이를 비교한다.

### 커버리지 갭 시각화

```
Waymo    : [████████░░░░░░] 58% coverage (26개 중 15개 카테고리 커버)
nuScenes : [██████░░░░░░░░] 42% coverage (26개 중 11개 카테고리 커버)
Lyft     : [███████░░░░░░░] 50% coverage

갭: "교량 위 역주행", "야간 보행자 돌발 횡단" 등 위험 시나리오 누락
```

## 장단점

**장점**
- 이기종 데이터셋의 시나리오 커버리지를 동일 기준으로 비교 가능
- 26개 분류 체계를 LLM 기반 분류에 직접 참고 가능
- MetaDrive 시뮬레이터와 통합되어 갭 시나리오 재현 가능

**단점**
- 자율주행 데이터셋(nuScenes 등)에 최적화 → 일반 캡션 텍스트에 바로 적용 불가
- 26개 분류가 고정되어 있어 도메인 특화 카테고리 추가 필요
- 원본 로그 데이터 필요 (캡션만으로는 사용 제한)

## 프로젝트 적용 포인트

### Gap-3 → EXP-002 Phase B (LLM 분류체계 설계 참고)

ScenarioNet의 **26개 시나리오 분류 체계**를 LLM 직접 시나리오 분류(3-3 방식)의 초기 taxonomy로 활용:

```python
# ScenarioNet 26개 카테고리를 GPT-4o 프롬프트에 주입
system_prompt = """
다음 26개 시나리오 카테고리를 참고하여 캡션을 분류하라:
교차로 직진, 교차로 좌회전, 교차로 우회전, 라운드어바웃,
고속도로 순항, 고속도로 병목, 차선 변경 좌, 차선 변경 우,
보행자 횡단(신호), 보행자 횡단(무단), 자전거 추월,
야간 주행, 우천 주행, 안개 주행, 적설 주행,
급제동 상황, 근접 추월, 역주행 차량, 교량/터널,
...
"""
```
